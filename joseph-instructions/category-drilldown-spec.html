SHAFIK: Implementation Notes (delete this section)

PHP Logic

// In your category template (e.g. celebrities-type.php):

$filter = isset($_GET['type']) ? sanitize_text_field($_GET['type']) : '';

if (empty($filter)) {
    // STEP 1: Show the sub-category grid (hardcoded HTML above)
    // NO database query. Just static cards linking to ?type=generator etc.
    get_template_part('partials/category-browse', 'type');
} else {
    // STEP 2: Show filtered results (your existing code)
    // This is the only place that hits the database
    $celebrities = get_celebrities_by_type($filter, $page, $per_page);
    get_template_part('partials/celebrity-results');
}

Same Pattern for ALL Categories

  • /celebrities/type/ → 5 type cards (above)
  • /celebrities/profile/ → 12 profile cards (above)
  • /celebrities/sun-sign/ → 12 zodiac cards (above). Same layout for Moon, Rising, MC, DC, IC
  • /celebrities/authority/ → 7 authority cards (above)
  • /celebrities/strategy/ → 4 strategy cards
  • /celebrities/definition/ → 6 definition cards (above)
  • /celebrities/incarnation-cross/ → ~192 cross cards (group by RAX/LAX/JXP)
  • /celebrities/variables/ → 16 variable combination cards

Key Rules

  • NEVER show a flat list of all celebrities. Always require at least one filter.
  • NO record counts displayed anywhere (no "34,521 Generators").
  • NO pagination showing total ("Page 1 of 600") — just "Next >" / "Previous".
  • Sub-category data is HARDCODED in the template. No DB query for the browse page.
  • Only the filtered results page (?type=generator) hits the database.
  • Remove the duplicate "Browse by Type" / "Type" sections in the sidebar.

Design Tokens

  • HD categories use gm-purple (#6E5898) accent
  • Astrology categories use gm-orange (#E8961D) accent
  • Cards: bg-gm-light rounded-xl with hover border + shadow
  • Grid: grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 (or 4 for zodiac)
  • Font: Inter / Bilo stack (same as Learn Hub nav)
